# レビュー文書 保守ガイド（定義の正典マップ）

本資料は、レビューガイドライン一式を **冗長にせず保守する** ための編集ルールである。各定義には「唯一の出所（正典）」が1か所だけあり、他の文書はそこを **参照リンク** で指す。共通概念を追記・修正するときは、まず本ガイドで正典を確認し、**正典だけを直す**こと。

> 背景: 大枠の完成後、「同じ定義・同じ文章が複数文書に多重掲載」されて冗長になっていた（一部の定義は6回出現）。2026-06-30 に各定義を正典へ一元化した。本ガイドはその状態を維持するためのもの。

---

## 1. 設計原則：Single Source of Truth

- 各事実（定義・図・表）は **正典1か所のみ** に置く。
- それ以外の文書では、その事実を **再掲せず**、正典への相対リンク＋「その文書固有の差分」だけを書く。
- 「便利だから」と他文書に定義表や図をコピーしない。重複の再発が冗長化の主因。

## 5. 編集後の検証（冗長の再発チェック）

リポジトリルートで以下を実行し、重複が正典1か所のみに収まっているか確認する。

```bash
# サイクル図は標準のみ
grep -rln "指摘 → 修正 → レビュアーが再確認" --include="*.md" | grep -v 資料/ | grep -v ddd
# 重大度フル定義（英語Must行）は標準のみ
grep -rln "| Must |" --include="*.md" | grep -v 資料/ | grep -v ddd
# 4層判断フロー図は総覧のみ
grep -rln "ツールで決定論的に検出できる" --include="*.md" | grep -v 資料/ | grep -v ddd
# 各ガイドラインに §4 旧見出し「**チェック内容**」が復活していないか（0件が正）
grep -rln "チェック内容" --include="*ガイドライン.md" .
```

加えて、相対リンク切れがないこと、各観点（A〜G）が「なぜ見るか(§4)・分類(仕分け§3)・操作チェック(§7)」のいずれかに必ず残っていることを確認する。
